CRITICAL PHASE 1 GAPS (4 Features = 80% Impact)
1. Natural Language Prompts (prompt)
   - Allows users to describe crawls in English
   - Firecrawl generates optimal parameters
   - Dramatically improves UX

2. Page Interaction Actions (scrapeOptions.actions)
   - 8 action types: wait, click, scroll, write, press, screenshot, executeJS, pdf
   - Essential for dynamic/interactive sites
   - Unlocks: pagination, lazy loading, SPAs

3. Browser Loading Controls (waitFor, timeout)
   - Critical reliability improvement
   - Prevents failures on slow/JS-heavy sites
   - Simple to implement, high impact

4. Response Pagination (.next field)
   - Handles large-scale crawls (>500 pages)
   - Currently unsupported
   - Critical for enterprise use cases
